01

Overview & Goals

The problem. SportsEngine bundles registration, scheduling, communication, and team management into a single expensive platform that does most things adequately and nothing exceptionally. As the organization grows, it creates friction for volunteer admins who manage too many disconnected workflows, and for parents who receive communication from too many places.

The approach. Replace SportsEngine with a purpose-built stack: Prizum Sports as the organization backbone (registrations, member directory, payments, facilities), GameChanger as the anchor for all game-related activity managed by Head Coaches, and a custom admin layer that aggregates GC team calendars into a single master org calendar. The solution is designed from day one to scale across multiple organizations using Prizum as the shared member and payment infrastructure.

Design principle: An admin should never need to open a second tab to understand the status of a player, event, or team. Every action that requires attention is surfaced with a clear next step — not just a status indicator.
Goal priorities ranked by impact
01 Reduce admin complexity
02 Improve parent experience
03 Minimize custom dev investment
04 Use GameChanger as game SoR
05 Scale to multi-org with Prizum as shared backbone
02

System Architecture

The recommended architecture is a Prizum-anchored hybrid stack designed for single-org deployment today and multi-org scale tomorrow. Prizum Sports owns the member directory, payments, and facility data across all organizations. Head Coaches manage their teams entirely within GameChanger. The Org Admin uses a custom layer to collect each team's GC iCal feed and merge them into a single master org calendar — the critical aggregation step that no off-the-shelf tool handles for this use case. Umpire scheduling is a planned future capability.

GC API limitation: GameChanger has no public API. All data flow to/from GC is via CSV import and iCal export. Head Coaches manage everything inside GC; the Org Admin only needs the team's iCal URL to connect it to the master calendar.
Multi-org scale model: Prizum Sports serves as the shared backbone across organizations — each org has its own member directory, payment processing, and facility management within Prizum. The custom admin layer and calendar aggregation logic are org-agnostic. Adding a new organization means provisioning a Prizum org, onboarding their GC teams, and connecting their iCal feeds — no code changes required.

Scheduling & Calendar
GC + Custom Build ▶

What's needed

  • Head Coaches manage team schedules entirely in GameChanger
  • Org Admin collects each team's GC iCal URL and registers it in the system
  • Custom calendar aggregates all team iCal feeds into one master org calendar
  • Organization-wide events layered on top (tryouts, clinics, field crew)
  • Conflict detection across fields and teams
  • Umpire scheduling — future roadmap item (Phase 4)

Recommended tools

GameChanger (per-team, coach-managed) Custom calendar aggregator (Org Admin tool) GC iCal feeds (one per team, read-only)

How it works

  • Each Head Coach manages their team schedule in GC — no change to their workflow
  • Org Admin opens "Manage Calendars" in the admin dashboard and pastes each team's GC iCal URL
  • System subscribes, polls, and merges all feeds into the master org calendar
  • Org-level events (tryouts, facility bookings from Prizum) layer on top
  • Umpire scheduling to be added in a future phase once core system is stable

Who Uses It & How

🗂️
Volunteer Admin
Board member · 5–10 hrs/week

Primary tools

  • Admin Dashboard (daily)
  • Prizum Sports (registrations, members, facilities)
  • GC + Custom (scheduling)

Core tasks

  • Monitor registration pipeline in Prizum Sports
  • Assign players to teams; export CSV roster to GC
  • Add each team's GC iCal URL to build the master calendar
  • Resolve scheduling conflicts across fields and teams
  • Send mass communications via Prizum Sports

Key UX need

  • Single view of "what needs attention today"
  • Bulk actions (assign 20 players at once)
  • Never requires 4 tabs to complete 1 task
⚾
Head Coach
Volunteer coach · game days + practices

Primary tools

  • GameChanger (game day)
  • Admin Dashboard (roster PDF)
  • GC + Custom calendar (practice schedule)

Core tasks

  • Own team roster, schedule, and lineup entirely in GC
  • Score games and track stats in GC
  • Communicate with parents via GC team chat
  • Access parent contact info (from Prizum-sourced roster)

Key UX need

  • GC is their only app — do not ask them to use another tool
  • Roster is pre-loaded by Org Admin at season start; coach manages from there
  • Exportable roster PDF for game day binder
👨‍👧
Parent / Guardian
Mobile-first · time-constrained

Primary tools

  • GameChanger app (game day)
  • Prizum Sports (registration)
  • TBD website (info lookup)

Core tasks

  • Register child for season
  • Watch live scoring in GC
  • RSVP to games and practices
  • Find schedule and field location
  • Receive org announcements

Key UX need

  • Registration under 5 min on mobile
  • One communication channel (not 3)
  • GC app for game-day engagement

Risks & Mitigations

HIGH

GameChanger CSV format changes or access restrictions

GC has no contractual obligation to maintain CSV export formats. If they change the format, your roster sync breaks. If they restrict exports, the process breaks entirely.

→ Mitigation: Always maintain your own database as the master record. Never rely on GC as a source of truth for player data. Run automated weekly exports to a backup store. GC is downstream-only.

HIGH

Volunteer adoption failure — too many new tools at once

Volunteer admins have limited time and low tolerance for learning new systems. Introducing 4 new platforms simultaneously will cause adoption collapse and reversion to spreadsheets.

→ Mitigation: Phase 1 should introduce exactly two new things admins must learn. Sequence carefully. Provide recorded walkthroughs (Loom), not manuals. Designate one "system owner" per tool.

MED

Admin dashboard scope creep in custom development

The admin dashboard is the highest-risk custom build. Teams often try to replicate functionality already provided by the SaaS tools, resulting in a bloated, fragile codebase.

→ Mitigation: Define dashboard scope tightly — it must aggregate and link, not rebuild. If a tool has a good UI, link to it; don't rebuild it. Dashboard = status + navigation, not feature replacement.

MED

Data drift between systems mid-season

With 5+ systems, player data can drift: a roster change in your DB doesn't automatically update GC, and a parent contact update in GC doesn't sync back. Admins end up with 3 versions of "the truth."

→ Mitigation: Assign one person as "data owner" responsible for sync checkpoints. Establish clear SoR policies (Prizum Sports owns player/contact/facility data; GC owns game data). Run sync audits monthly.

MED

Prizum Sports vendor dependency

Consolidating registrations, member management, and facility management into Prizum Sports creates a meaningful single-vendor dependency. If Prizum Sports raises prices, changes API access, or sunsets features, three functional areas are impacted at once.

→ Mitigation: Maintain your own data layer with full export capability from Prizum Sports. Confirm API access terms in contract. Never store data exclusively in their platform — keep regular exports. The flip side: consolidation also reduces integration points and operational risk.

LOW

iCal feed management as org scales

As team count grows (or across multiple orgs), maintaining a growing list of manually-added GC iCal URLs becomes an admin burden. A missed or expired URL means a team's games disappear from the master calendar silently.

→ Mitigation: Build iCal URL management as a first-class admin feature — with status indicators showing last successful sync, stale feed warnings, and a simple "add team feed" flow. Consider automated staleness alerts.

LOW

Parent communication fragmentation

Parents currently receive messages from SportsEngine, GC, email, and text. The new system could replicate this fragmentation if communication channels aren't rationalized.

→ Mitigation: Establish clear channel policies on day one: GC for game-day and team chat; Prizum Sports for org-wide announcements and registration-related messages. Communicate the policy to parents at season start.
